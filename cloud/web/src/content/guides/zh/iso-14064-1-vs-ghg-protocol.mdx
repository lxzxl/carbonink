问三位从业者"企业碳盘查该按什么标准做"，两个名字都会被提到。诚实的答案是：
ISO 14064-1 和 GHG Protocol 与其说是竞争关系，不如说是干两件不同事情的
两份文件 —— 而且大多数认真做的清单，实际上两个都在用。下面把区别讲透，
不带供应商滤镜。

## 两份文件，两种职能

**GHG Protocol 企业核算与报告标准**（WRI/WBCSD 出品，免费下载）是一套
*方法学*：教你怎么定边界、怎么理解范围一/二/三、怎么算、按什么原则报告。
全球主流披露体系 —— CDP、SBTi、CSRD 下的 ESRS、美国证监会与加州的气候
规则 —— 的核算语言都源自它。

**ISO 14064-1:2018**（付费的 ISO 标准文本）是一套*要求性标准*：一张可审计
的"shall"清单，规定合格的清单和报告必须包含什么。它的价值在于与
**ISO 14064-3**（核查标准）形成的接口 —— 当招标方、监管者或客户要求
"按 ISO 14064-1 编制并经第三方核查"时，这个要求是可检验的；"我们参照了
良好实践"则不可检验。

## 逐项对比

| | ISO 14064-1:2018 | GHG Protocol 企业标准 |
| --- | --- | --- |
| 发布方 | ISO | WRI + WBCSD |
| 获取 | 付费标准 | 免费 PDF |
| 性质 | 可认证的要求性标准（"shall"） | 方法学 + 指南 |
| 排放结构 | 直接排放（类别 1）+ 五类间接排放（类别 2–6） | 范围 1、2、3（范围三细分 15 类） |
| 核查接口 | 与 ISO 14064-3 配套设计 | 无配套鉴证标准 |
| 范围二处理 | 类别 2；实务中常按 GHGP 指南双轨报告 | 明确要求 location-based + market-based 双轨 |
| 典型使用场景 | 招标、认证、亚洲监管（如台湾上市公司强制盘查）、供应链"按 ISO 核证"要求 | CDP、SBTi、CSRD/ESRS、多数自愿披露 |

## 范围与类别怎么对应

两套结构可以干净地互相折算，任何方向都不丢信息：

| GHG Protocol | ISO 14064-1:2018 |
| --- | --- |
| 范围一 | 类别 1（直接排放） |
| 范围二 | 类别 2（输入能源） |
| 范围三·运输相关（15 类中的第 4、6、7、9 类） | 类别 3（运输） |
| 范围三·采购的货物与服务、资本货物、燃料与能源相关、废弃物（第 1、2、3、5 类） | 类别 4（组织使用的产品） |
| 范围三·下游：售出产品的使用与报废、租赁、特许经营、投资（第 10–15 类） | 类别 5–6 |

数据模型建得好，每个排放源只录一次，两种口径都能渲染出来 —— 软件就该这么
处理两套标准（例如碳墨的报告导出按 ISO 14064-1 结构组织，同时附范围映射）。

## 什么时候该以 ISO 14064-1 为框架

- 客户、招标文件或认证体系**点名 ISO 14064-1** —— 在东亚供应链和政府采购里
  很常见。
- 计划做**第三方核查**，希望清单直接按核查员将来要对照的要求（ISO 14064-3）
  来搭。
- 所在地监管直接引用 ISO —— 台湾对上市柜公司分阶段强制盘查是当下最典型的
  例子。

## 什么时候该以 GHG Protocol 为框架

- 披露目标是 **CDP、SBTi 或 CSRD/ESRS**，它们的问卷和数据点原生说"范围"
  语言。
- 需要**免费且够细的计算指引** —— 范围二指南和 15 类范围三标准的"菜谱级"
  细节，ISO 体系里没有对应物。
- 刚起步，想先用最通行的词汇跟同行对标。

## 实务答案：两个都用

严肃清单的标准做法是三层：

1. **用 GHG Protocol 方法学计算** —— 定边界、范围二双轨、按 15 类筛查范围三。
2. **按 ISO 14064-1 组织和声明报告** —— 类别 1–6、重要性判定准则、
   不确定性、符合性声明。
3. 下游有鉴证需求时，**按 ISO 14064-3 核查**。

这样做没有重复劳动：一套数据、两种呈现，无论对面是 CDP 填报口还是在招标书里
引用 ISO 的采购官，都能拿到他们要的形状。完整编制流程见
[ISO 14064-1 清单编制七步](/zh/guides/ghg-inventory/)。
