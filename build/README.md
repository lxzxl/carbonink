# `build/` — electron-builder resources

This directory holds the resources electron-builder consumes when
producing distributable installers (`.dmg`, `.exe`).

## Contents

| File                     | Purpose                                                    |
| ------------------------ | ---------------------------------------------------------- |
| `icon.png`               | 1024×1024 PNG fallback icon                                |
| `icon.icns`              | macOS app icon (Apple Icon Image)                          |
| `icon.ico`               | Windows app icon (multi-resolution ICO)                    |
| `entitlements.mac.plist` | macOS Hardened Runtime entitlements (required to notarize) |
| `sign-windows.js`        | electron-builder hook → Azure Trusted Signing via signtool |

## ⚠️ Placeholder icons

The icons currently shipped here are **placeholders** — a flat blue tile
with a single white "C" glyph, written as raw PNG bytes inside the
`.icns` and `.ico` filenames so that electron-builder's path-existence
check passes during config validation. They were generated by
`scripts/generate-placeholder-icons.mjs`.

**Before shipping a signed production build:**

1. Replace `icon.png`, `icon.icns`, and `icon.ico` with real branded
   assets at the formats electron-builder expects:
   - `icon.icns` — Apple Icon Image container with 16/32/64/128/256/512/1024 layers
   - `icon.ico` — Windows ICO container with 16/24/32/48/64/128/256 layers
   - `icon.png` — 1024×1024 PNG (Linux + generic fallback)
2. Delete `scripts/generate-placeholder-icons.mjs` (optional — keeping
   it lets future contributors regenerate placeholders for prototype
   builds).
3. Update this README to remove the placeholder warning.

A production `dist:*` run with these placeholders will likely produce
broken-looking installer icons, but the build itself will succeed.

## Why not `.gitignore` the icons?

Earlier the project ignored `build/` wholesale. Phase 5 inverts that:
the icon stubs are committed so CI builds don't need a separate
generation step. The real production icons will simply overwrite the
placeholders in a future commit.
